## 🌟 项目概述

**NoteKlip** 是一个隐私优先、基于浏览器的 Kindle 标注管理与可视化仪表盘。只需上传你的 `My Clippings.txt` 文件，即可立即深入洞察你的阅读习惯，发现跨书籍和作者的模式，并重温你最喜欢的段落——所有数据完全本地处理，不上传任何服务器。

> **🔒 隐私承诺：** 所有处理都在你的浏览器本地完成。零数据收集，零追踪，100% 属于你。

## 🚀 快速开始

### 环境要求

开始之前，请确保已安装以下软件：

- **Node.js** `18.0` 或更高版本 ([下载](https://nodejs.org/))
- **npm** `9.0+` 或 **yarn** `1.22+`
- 现代浏览器（Chrome、Firefox、Safari 或 Edge）

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/UNICKON/klips.git
cd klips

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

🎉 **成功！** 访问 `http://localhost:5173` 即可体验 NoteKlip。

### 生产构建

```bash
# 创建优化的生产构建
npm run build

# 本地预览生产版本
npm run preview
```

优化后的文件将位于 `dist/` 目录中，可直接部署。

## 📦 部署指南

### 方案一：Cloudflare Pages（推荐）

**为什么选择 Cloudflare Pages？**
- ⚡ 超快的全球 CDN
- 🆓 免费托管，无限带宽
- 🔄 推送自动部署
- 🌐 支持自定义域名

**配置步骤：**

1. **连接仓库**  
   - 访问 [Cloudflare Pages](https://pages.cloudflare.com/)
   - 点击"创建项目" → 连接到 GitHub
   - 选择 `klips` 仓库

2. **配置构建设置**
   ```yaml
   构建命令: npm ci && npm run build
   输出目录: dist
   环境变量:
     VITE_BASE: /
   ```

3. **部署**  
   - 点击"保存并部署"
   - 每次推送到 `main` 都会自动部署
   - 访问地址：`https://<你的项目>.pages.dev`

4. **自定义域名**（可选）
   - 进入项目 → 自定义域
   - 添加你的域名（如 `noteklip.org`）
   - 按提示更新 DNS 记录

### 方案二：GitHub Pages

**配置步骤：**

1. **启用 GitHub Pages**
   - 进入仓库 `Settings` → `Pages`
   - 来源：选择 **GitHub Actions**

2. **自动部署工作流**  
   内置的 `.github/workflows/pages.yml` 会自动处理：
   - 每次推送到 `main` 时构建
   - 自动设置正确的基路径
   - 部署到 GitHub Pages

3. **访问你的站点**
   - 访问：`https://<用户名>.github.io/klips/`
   - 首次部署需要 2-3 分钟

> **💡 提示：** 项目通过 `VITE_BASE` 环境变量自动检测部署路径，因此路由和资源在两个平台上都能正常工作。

## 📖 使用指南

### 1️⃣ 导入你的标注

**步骤 1：从 Kindle 导出**
1. 通过 USB 将 Kindle 设备连接到电脑
2. 进入 `Kindle 驱动器` → `documents` → `My Clippings.txt`
3. 将文件复制到电脑

**步骤 2：导入到 NoteKlip**
1. 在浏览器中打开 NoteKlip
2. 进入 **⚙️ 设置** 页面
3. 点击 **导入标注** 部分
4. 选择一种方法：
   - **📁 文件上传**：点击"选择文件"并选择 `My Clippings.txt`
   - **📋 直接粘贴**：将文件内容复制粘贴到文本区域
5. 点击 **导入** 并等待处理

> **💡 提示：** 你可以多次导入。NoteKlip 会自动检测并跳过重复的高亮。

### 2️⃣ 探索你的数据

**仪表盘 - 鸟瞰视图**
- 通过交互式图表查看随时间变化的阅读趋势
- 识别你最高效的阅读时段
- 发现哪些书籍和作者最吸引你
- 拖放卡片自定义仪表盘布局

**书库 - 书籍集合**
- 在可视化网格中浏览所有高亮的书籍
- 按标题或作者名称搜索
- 按添加日期、标题或高亮数排序
- 点击任何书籍查看其所有高亮

**高亮 - 深入探索**
- 即时搜索所有高亮
- 按作者筛选以专注于特定作家
- 一键复制任何高亮用于笔记
- 查看原始页码和时间戳

**导出 - 数据所有权**
- 将所有数据备份为 JSON 文件
- 从以前的备份恢复
- 在设备之间传输数据
- 完全控制你的阅读历史

### 3️⃣ 自定义你的体验

**个性化仪表盘**
- 拖动任何图表卡片重新排序
- 你的布局会自动保存
- 随时重置为默认布局

**语言切换**
- 点击侧边栏的语言切换按钮
- 在中英文之间即时切换
- 所有界面文本立即更新

**持久化偏好设置**
- 所有设置都保存在本地
- 你的偏好在会话之间持久保存
- 无需账户，无需云同步

## 🔒 隐私与安全

NoteKlip 以隐私为基础构建。这是我们的承诺：

### ✅ 我们做什么

- **100% 本地处理**：所有数据都留在你的浏览器中
- **零服务器上传**：你的高亮永不离开你的设备
- **无追踪**：无分析、无 Cookie、无遥测
- **开源**：GitHub 上完全透明的代码
- **离线可用**：首次加载后无需互联网
- **数据可移植**：永久导出和拥有你的数据

### ❌ 我们不做什么

- ❌ 无用户账户或身份验证
- ❌ 无云存储或同步
- ❌ 无第三方服务或 API
- ❌ 无广告或商业化
- ❌ 无数据收集或分析
- ❌ 无追踪像素或指纹识别

### 🔐 数据存储方式

| 存储类型 | 用途 | 持久性 |
|-------------|---------|-------------|
| **IndexedDB** | 高亮与书籍 | 永久（直到你清除） |
| **LocalStorage** | 用户偏好 | 永久（直到你清除） |
| **SessionStorage** | 临时 UI 状态 | 标签页关闭时清除 |

**你始终拥有控制权。** 随时通过浏览器设置或内置清除功能清除你的数据。
